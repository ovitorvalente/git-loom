# Git Loom - Branding

## Nome e posicionamento

- Nome: `gitloom`
- Pronuncia: "git-lum"
- Categoria: CLI de produtividade para Git
- Tagline principal: `Revise melhor antes de commitar`
- Tagline alternativa: `Planeje melhor seus commits`

## Essencia da marca

`gitloom` existe para tornar o fluxo de commit mais claro, previsivel e profissional.
O produto nao tenta esconder o Git. Ele atua como uma camada de planejamento e revisao
entre o estado do working tree e a decisao final de commitar.

O posicionamento correto nao e "magia". E "controle com assistencia".

## Promessa do produto

- agrupa mudancas em blocos pequenos
- gera mensagens semanticas em portugues
- mostra score, alertas e sugestoes antes de commitar
- reduz commits vagos e acidentais
- permite automacao com saida JSON
- ajuda o usuario a decidir antes de executar

## Valores da marca

- Clareza: o usuario deve entender o que sera commitado
- Precisao: a ferramenta deve evitar generalidades desnecessarias
- Confianca: o fluxo deve ser previsivel, revisavel e testavel
- Pragmatismo: o produto ajuda sem tirar controle
- Portabilidade: roda bem em Linux, macOS e Windows

## Personalidade

- tecnica
- direta
- calma
- profissional
- sem exagero visual

`gitloom` nao deve soar como brinquedo, bot promocional ou ferramenta "muito esperta".
Ele deve parecer uma ferramenta confiavel de engenharia.

## Direcao visual

### Cores

Paleta sugerida para materiais visuais, site e screenshots:

- Primaria: `#0F766E`
- Primaria escura: `#115E59`
- Destaque: `#0EA5E9`
- Sucesso: `#16A34A`
- Aviso: `#D97706`
- Erro: `#DC2626`
- Fundo claro: `#F8FAFC`
- Texto forte: `#0F172A`
- Texto secundario: `#475569`

Evitar:

- roxo como cor dominante
- visual neon
- contraste fraco
- excesso de gradientes chamativos

### Tipografia

- Headlines: `IBM Plex Sans`, `Manrope` ou `Sora`
- UI/Body: `IBM Plex Sans` ou `Inter`
- Monospace: `JetBrains Mono` ou `IBM Plex Mono`

### Iconografia

O simbolo deve transmitir:

- fluxo
- tecelagem
- composicao
- estrutura

Direcoes validas:

- linhas entrelacadas
- malha modular
- nodo central conectando mudancas
- grafico/tecido abstrato

Evitar:

- martelo
- foguete
- robo simpatico
- emoji como logo principal

## Tom de voz

### Bom

- objetivo
- tecnico
- orientado a decisao
- sem marketing inflado
- sem prometer inteligencia opaca

### Evitar

- "revolucionario"
- "magico"
- "sem esforco"
- "turbinado por IA" como centro da mensagem

## Mensagem curta de produto

`gitloom` e um CLI em Go para revisar, agrupar e criar commits semanticos com feedback acionavel antes da execucao.

## Descricao curta para GitHub

`CLI em Go para revisar e criar commits semanticos com agrupamento inteligente, score de qualidade e saida JSON.`

- `golang`

## Casos de uso que a marca deve reforcar

- revisar antes de commitar
- padronizar mensagens em times
- automatizar pipelines e scripts com `--json`
- validar ambiente com `doctor`
- melhorar qualidade de historico Git
- reduzir commits grandes e pouco claros

## Assinatura visual de output

O output do CLI deve continuar:

- escaneavel
- modular
- orientado por secoes
- com score visivel
- sem parecer log bruto

O terminal e parte central da identidade do produto.
